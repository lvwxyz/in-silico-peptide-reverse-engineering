# [01] 듀얼 스트림(Dual-Stream) 기반 융합 아키텍처 및 데이터 파이프라인 설계

## 📌 작업 목표

1D 후성유전 서열 데이터와 2D 공간 전사체(Spatial Transcriptomics) 데이터를 결합하여 대식세포의 M1 분극화(염증 반응)를 예측하는 핵심 딥러닝 파이프라인(Block 1~3)을 설계했습니다. 이질적인 두 차원의 데이터를 효과적으로 융합하고 노이즈를 필터링하는 것이 주 목적이었습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **다중 오믹스 데이터 소스 연동:** TCGA(Melanoma), GEO(GSE86940), ENCODE 데이터베이스를 전처리하여 입력 벡터화.
    
- **공간 데이터 연동:** 10x Genomics Visium 데이터의 $(x, y)$ 좌표와 RNA 발현량을 활용하여 반경 50$\mu m$ 이내 세포 간 물리적 거리를 GNN 인접 행렬(Adjacency Matrix)로 변환 (Squidpy, PyTorch Geometric 활용).
    
- **듀얼 스트림 특징 추출기 구축:** - 서열 문맥 캡처: 양방향 분석을 위한 `Bi-LSTM` 레이어 적용.
    
    - 공간 상호작용 캡처: 세포 간 군집 분석을 위한 `GraphSAGE` 레이어 적용.
        
- **VAE 기반 잠재 공간 융합:** 두 스트림에서 나온 텐서를 Concatenate한 뒤, 변이형 오토인코더(VAE)의 Reparameterization Trick을 사용해 노이즈가 필터링된 가우시안 잠재 공간(Latent Space) 벡터로 맵핑.
    

## 💻 핵심 코드 스니펫

Python

```
# Block 3: VAE 융합 코어 (이질적 데이터 결합 및 노이즈 필터링)
def forward(self, seq_vec, spatial_vec):
    # 1D 서열 특징(Bi-LSTM)과 2D 공간 특징(GNN) 결합
    merged_vec = torch.cat([seq_vec, spatial_vec], dim=1)
    
    # 잠재 벡터(Latent Vector) 추출
    mu = self.fc_mu(merged_vec)
    logvar = self.fc_logvar(merged_vec)
    
    # Reparameterization Trick 적용
    std = torch.exp(0.5 * logvar)
    eps = torch.randn_like(std)
    z = mu + eps * std
    
    # 염증 확률 예측 (Classifier)
    inflammation_prob = self.classifier(z)
    return inflammation_prob, mu, logvar
```

## ❌ 실패하거나 버려진 시도

- **Ubiquitination 및 Phosphorylation 마커 사용 포기:** 초기에는 모든 후성유전 마커를 피처로 사용하려 했으나, ChIP-seq 데이터 특성상 결측치가 심각해 모델 설명력(Explainability)이 훼손될 위험이 컸음. 생물학적 기전이 확실하고 결측치가 적은 메틸화(H3K4me3, H3K27me3)와 아세틸화(H3K27ac) 마커로 피처를 축소/개편함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

모델의 핵심 진단 코어(염증 발생 확률 예측)가 완성되었습니다. 하지만 사용자가 '초기 후성유전 시퀀스'를 직접 입력해야 하는 구조는 in vitro 실험 없이 불가능하다는 입력 병목 현상이 발견되어, 생체재료 물성을 서열로 번역해 주는 입력부(Block 0)의 설계로 넘어갔습니다.

# [02] 입력부(Block 0) 하이브리드 개편: 펩타이드 수식 생체재료 도입

## 📌 작업 목표

일반 사용자가 칩시퀀싱(ChIP-seq) 데이터 없이도 생체재료의 특성만으로 모델을 구동할 수 있도록, 재료 특성을 초기 후성유전 시퀀스로 추론/번역하는 입력부 맵핑 모듈(Block 0)을 구축했습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **하이브리드 입력 체계 도입:** 단순 물리적 물성(탄성도, 거칠기 등) 입력 방식을 폐기하고, "물성(Base Material) + 표면 펩타이드 서열(Surface Peptide, 예: CD47, RGD)"의 조합을 입력받도록 시스템 전면 개편.
    
- **기계적 형질도입(Mechanotransduction) 룰셋 정의:** 인테그린 결합 및 TLR4 수용체 자극 기전에 따라 특정 펩타이드가 히스톤 아세틸화/메틸화 스위치를 어떻게 변경하는지 Look-up Table 기반 초기 시퀀스 번역 매커니즘 구축.
    
- **초기 시퀀스 생성 자동화:** 염증 유전자(TNF-$\alpha$, IL-6 등)의 프로모터 뼈대 서열에 예상 마커를 자동 합성(예: `G` $\rightarrow$ `G_ac`)하여 1D 입력 벡터로 전달하는 로직 추가.
    

## ❌ 실패하거나 버려진 시도

- **단순 물리화학적 특성 기반의 예측(Look-up Table):** 재료의 거칠기나 탄성도만으로 서열을 예측하려는 시도는 예측 결과가 지나치게 단순화(Oversimplification)되어 생물정보학적 가치가 떨어진다고 판단, '표면 펩타이드 서열'이라는 화학적/미시적 시퀀스 데이터를 Input 차원에 추가함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

AI가 연산할 수 있는 명확한 트리거(예상 초기 시퀀스)가 확보되었습니다. 예측 모듈을 역으로 돌려 '염증이 억제되는 최적의 조건'을 찾는 역설계 엔진(GA Optimizer)을 구축하는 단계로 진행했습니다.

# [03] 유전 알고리즘(GA) 기반 역설계 엔진 및 비용 함수 설계

## 📌 작업 목표

M1 대식세포 분극화(염증) 확률을 사용자가 설정한 목표치(예: 15% 미만) 이하로 낮추기 위해, 가장 적은 마커 조작으로 최적의 후성유전 시퀀스를 찾아내는 진화 연산 기반의 생성형 역설계 엔진(Block 4)을 개발했습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **가상 시나리오 동시 병렬 처리:** 초기 세대(초기 예상 시퀀스)를 기반으로 변이 마스크(Mutation Mask)를 씌워 가상 시퀀스 배치를 생성하고 병렬 연산.
    
- **정교한 적합도(Fitness) 함수 설계:**
    
    - `Inflammation`: Block 3에서 산출된 염증 발생 확률.
        
    - `MutationPenalty`: 실제 생체재료 제조의 실현 가능성을 반영한 패널티. 초기 서열과 변이 서열 간의 불일치 비율(Hamming Distance)을 계산.
        
    - 가중치 $\alpha$(염증 점수 통제력)와 $\beta$(조작 패널티 통제력) 파라미터를 추가하여 Trade-off(Pareto Front)를 조절할 수 있도록 고도화.
        

## 💻 핵심 코드 스니펫

Python

```
def compute_fitness(self, mutated_seq, original_seq, spatial_graph):
    # 1. 염증 확률 계산 (GNN + Bi-LSTM + VAE 융합 코어 통과)
    seq_vec = self.seq_extractor(mutated_seq)
    spatial_vec = self.spatial_extractor(spatial_graph.x, spatial_graph.edge_index)
    pred_prob, _, _ = self.fusion_core(seq_vec, spatial_vec)
    inflammation = pred_prob.item()
    
    # 2. 마커 변이율 (Manufacturability Penalty) - Hamming Distance 기반
    mutation_count = torch.sum(mutated_seq != original_seq).item()
    mutation_penalty = mutation_count / original_seq.size(1) 
    
    # 최종 비용 함수: 염증이 낮고 변이율이 낮을수록 높은 Fitness 반환
    fitness = 1.0 / (self.alpha * inflammation + self.beta * mutation_penalty + 1e-6)
    return fitness, inflammation
```

## ❌ 실패하거나 버려진 시도

- **마커 변경 절대 개수(L1 Norm) 기반 패널티 부여:** 절대값으로 계산할 경우 시퀀스 길이가 달라지면 기준이 흔들리는 문제가 있어, 시퀀스 전체 길이 대비 변경된 비율(Percentage)로 연산 구조를 변경함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

역설계 모듈의 수학적, 알고리즘적 토대가 완성되어 '마커 변이율 15% 이하 유지'라는 정량적 목표(KPI) 통제가 가능해졌습니다. 마지막으로, 생성된 1D 시퀀스가 어떻게 2D 조직 상의 주변 세포들에게 파급력을 미치는지 연산하는 '가상 이식' 메커니즘을 구체화했습니다.

# [04] 가상 이식(Virtual Implantation) 맵핑 및 GNN Message Passing 구현

## 📌 작업 목표

입력받은 1D 시퀀스(생체재료 자극)가 조직 내 특정 세포에 이식되었을 때, 주변 세포들로 염증 신호가 전파되는 주변분비 신호전달(Paracrine Signaling) 과정을 GNN을 통해 시뮬레이션했습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **10x Visium 기반 레퍼런스 맵 활용:** 모델이 무에서 공간을 생성하는 것이 아니라, 이미 다운로드된 Visium 공간 전사체 데이터를 '가상 피부 조직 테스트 베드'로 설정.
    
- **Node Feature Overwrite (세포 자극 초기화):** 조직 내 이식 부위 경계면(Interface)에 해당하는 대식세포 타겟 노드들의 특징(Node Feature)을 AI가 생성한 '초기 후성유전 1D 벡터'로 강제 주입(Mapping).
    
- **Message Passing 연산:** 타겟 노드의 변형된 신호가 간선(물리적 거리 반경)을 타고 인접한 섬유아세포/림프구 노드로 전달되어 전체 네트워크가 업데이트되도록 GraphSAGE 계층 활용.
    
- **Global Mean Pooling:** 노드 단위 연산 후, 그래프 전체를 하나의 벡터로 압축하여 조직 전체 스케일의 최종 염증 확률(Diagnosis)을 집계.
    

## ❌ 실패하거나 버려진 시도

- **서열 데이터로부터 좌표를 생성하려는 접근:** 초기 설계 시 1D 서열 정보에서 공간 좌표를 유추하려 했으나, 이는 수학적/생물학적으로 성립할 수 없는 인과율 오류임이 파악됨. 10x Visium 데이터를 베이스 캔버스로 고정하고, 그 위에 Feature를 맵핑하는 'Virtual Implantation' 방식으로 연산 파이프라인을 전면 수정.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

세포 단일 단위가 아닌, 기하학적 공간에서의 '조직(Tissue) 레벨 면역 상호작용'을 성공적으로 딥러닝(GNN)으로 치환했습니다. 최종적인 In Silico Reverse Engineering 텐서 연산 파이프라인이 완성되었으며, 민감도 최적화(Hyperparameter Tuning) 진행 단계로 돌입했습니다.
