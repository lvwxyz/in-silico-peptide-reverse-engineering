# [1] IEDB 기반 데이터 파이프라인 및 생물학적 임베딩 체계 구축

## 📌 작업 목표

생체재료 코팅용 펩타이드의 면역 거부반응(FBR)을 인공지능이 학습할 수 있도록, IEDB(Immune Epitope Database)의 T-cell Assay 데이터를 정제하고 기계학습용 이진 벡터로 변환하는 전처리 파이프라인을 구축함.

## 🔧 주요 변경 사항 / 시도한 접근법

- **길이 및 노이즈 필터링:** 11,025개의 Raw data 중 서열 길이 8~15mer만 필터링하고 비표준 아미노산 서열을 제거하여 최종 10,814개의 유효 데이터 확보.
    
- **클래스 통합:** 데이터의 Qualitative Measure를 기준으로 Positive 파생형들을 '1(염증 유발)'로, Negative를 '0(안전)'으로 통합. (최종 비율 약 1:1 대칭 확인)
    
- **BLOSUM62 기반 임베딩:** 단순 알파벳 원핫 인코딩이 아닌, 아미노산 간의 진화적/물리화학적 치환 유사성을 반영하기 위해 BLOSUM62 치환 행렬 가중치 적용.
    
- **Pre-padding 규격 설정:** 순차적 구조를 가진 모델(LSTM)의 정보 유실을 방지하기 위해 빈 공간을 서열 앞쪽(Pre-padding)으로 채워 고정된 15-mer 텐서로 통일.
    

## 💻 핵심 코드 스니펫

Python

```
def encode_peptide_blosum(sequence, max_len=15):
    # BLOSUM62 매트릭스 활용 및 Pre-padding 적용 (LSTM 정보 보존 목적)
    padded_seq = sequence.rjust(max_len, 'X')
    encoded = [blosum62_matrix.get(aa, default_val) for aa in padded_seq]
    return torch.tensor(encoded, dtype=torch.float32)
```

## ❌ 실패하거나 버려진 시도

- 극초기 구상에서 Multi-omics 및 Spatial Data를 펩타이드 정보와 병합하려 했으나, _in silico_ 모델링 시 데이터 결측치 통제와 구현 가능성(Implementability) 한계로 초기에 배제함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

1D 서열 기반의 정제된 입력 벡터 구축을 완료함. 실험 조건(하이드로젤 코팅 부착 등)에 따른 극단적 데이터 불균형 상황(Safe 1 : Inflam 9)을 상정한 Loss Function 보정 및 실제 모델 아키텍처 구축 단계로 이동.

# [2] 하이브리드 멀티모달 설계(Pipeline A) 및 데이터 편향 제어

## 📌 작업 목표

단순 서열 탐색을 넘어 국소적 모티프(Motif)와 전체 서열의 상호작용 맥락을 파악하기 위한 최적의 신경망 구조를 탐색하고, 소수 클래스 데이터 쏠림에 대응하는 구조 설계.

## 🔧 주요 변경 사항 / 시도한 접근법

- **BCEWithLogitsLoss 동적 가중치 계산:** 학습 중 `neg_weight` (소수 클래스 틀림에 대한 페널티)를 배치(Batch) 비율에 따라 동적으로 산출하여 편향 학습 원천 차단.
    
- **1D-CNN + Bi-LSTM 하이브리드 결합 (Early Fusion):**
    
    - `1D-CNN` (필터 32~64개, 커널 3): 염증을 유발하는 국소 에피토프 패턴(hCD47 등) 스캐닝.
        
    - `Bi-LSTM` (Hidden size 64): 아미노산 서열 전후의 양방향 맥락 정보 압축.
        
- **Ablation Study (모델 소거 테스트) 진행:** 1D-CNN 단일 모델부터 Attention 추가 모델까지 총 10여 개 모델을 평가.
    
- **Cascading Error 해결:** 깊은 네트워크 층에서 발생하는 역전파 소실을 잔차 연결(Residual Connection)과 어텐션 기법을 도입하여 방어.
    

## 💻 핵심 코드 스니펫

Python

```
# 소수 클래스 편향 학습을 방지하기 위한 동적 가중치 손실 함수
neg_weight = torch.tensor([neg_count / pos_count], device=device)
criterion = nn.BCEWithLogitsLoss(pos_weight=neg_weight)

# 하이브리드 포워드 및 잔차 연결(Residual Connection)
cnn_out = self.cnn(embedded_seq)
lstm_out, _ = self.bi_lstm(cnn_out)
attended_out = self.attention(lstm_out)
final_out = attended_out + lstm_out  # 기울기 소실 방지
```

## ❌ 실패하거나 버려진 시도

- 단순 CNN 구조만 차용했을 때, 거리가 먼 아미노산 간의 상호작용(예: 서열 양 끝단이 접히며 결합하는 현상)을 모델이 인지하지 못해 분류 정확도가 0.6 초반대에서 정체됨.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

최종 모델 평가 지표 AUROC 0.7235 달성. 이를 유전 알고리즘 역설계 루프에서 '면역 반응성'을 평가하는 심사위원(Predictor) 모델로 확정.

# [3] 유전 알고리즘(GA) 기반 펩타이드 역설계 최적화 (Pipeline B)

## 📌 작업 목표

AI 예측 모델을 활용해, 대식세포의 FBR(이물 반응) 확률을 최소화하면서 체내 적합성(수용성, 전하량) 및 타겟 모티프를 유지하는 최적의 펩타이드를 역탐색하는 루프 구축.

## 🔧 주요 변경 사항 / 시도한 접근법

- **다목적 적합도 함수(Fitness Function) 파라미터화:** - FBR 염증 억제 확률 (비중 0.5)
    
    - 체내 수용성 GRAVY Score 조절 (비중 0.15)
        
    - 펩타이드 전하량(Charge) 통제 (비중 0.15)
        
    - `EVTELT` 등 타겟 단백질(hCD47)과의 상동성 유지 (비중 0.2)
        
- **적응형 돌연변이(Adaptive Mutation) 도입:** 진화가 진행될수록(세대가 지날수록) 돌연변이 확률을 유동적으로 조정하여 다양성 유지.
    

## 💻 핵심 코드 스니펫

Python

```
def calculate_fitness(peptide_tensor):
    # Pipeline A의 AI 모델을 통한 염증 확률 예측
    p_fbr = ai_predictor(peptide_tensor) 
    
    # 화학적 제약조건 수치화
    gravy = calculate_gravy(peptide)
    homology = calc_homology(peptide, target_motif="EVTELT")
    
    # 페널티가 통합된 최종 비용(Cost) 함수
    fitness = (0.5 * (1 - p_fbr)) + (0.15 * gravy) + (0.2 * homology) - len_penalty
    return fitness
```

## ❌ 실패하거나 버려진 시도

- **조기 수렴(Local Optimum):** 초기 세대 평가 시, 상위 1등 개체에게만 교배 기회를 몰아주어(Selection Pressure 과다) 10세대 만에 50개 펩타이드 서열이 획일화되는 현상 발생. 이후 파레토 프론트(Pareto Front) 확인 후 토너먼트 선택압을 완화함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

총 30세대의 진화 루프를 통해 $P_{FBR}$ 확률 8.0% (염증 억제율 92%)의 최적 서열 `PCCCREVTELTKMCS` 도출 성공. 단, 파이썬 반복문에 의한 연산 병목 해결 필요성 대두.

# [4] Smart Router 엔진 도입 및 벡터 연산 최적화 (Global Optimization)

## 📌 작업 목표

서열 길이 변동 시 발생할 수 있는 잉여 연산 부하를 억제하고, 파이썬 중첩 루프에 의한 GA 진화 과정의 시간 지연(Bottleneck) 타개.

## 🔧 주요 변경 사항 / 시도한 접근법

- **서열 길이 기반 분기 스위치(Smart Router):** 15-mer 이하 폴리펩타이드는 가벼운 '1D-CNN Only 모델'로, 초과 서열은 'Hybrid (CNN+LSTM) 모델'로 넘기는 동적 라우팅 아키텍처 적용.
    
- **루프 벡터화(Vectorization):** GA 개체군 50개의 적합도 평가 시 문자열을 하나씩 for문으로 처리하던 방식을 폐기. 50개 서열을 하나의 `torch.Tensor` 배치로 묶어(Stack) GPU 연산을 1회만 호출하도록 수정.
    

## 💻 핵심 코드 스니펫

Python

```
def smart_router(peptide_batch_tensor):
    # 배치 텐서의 시퀀스 길이에 따른 가속 라우팅 알고리즘
    seq_length = peptide_batch_tensor.shape[1]
    
    if seq_length <= 15:
        return cnn_engine(peptide_batch_tensor)  # 고속 스캐닝
    else:
        return hybrid_engine(peptide_batch_tensor) # 심층 문맥 분석
        
# for 루프 제거 -> 병렬 텐서 연산 1회 수행
predictions = smart_router(torch.stack(population_encoded))
```

## ❌ 실패하거나 버려진 시도

- 단일 하이브리드 모델 구조 고집. 짧은 서열 평가 시 무의미한 LSTM 시퀀스 연산이 발생하여, Ablation Study 기준 실행 시간이 비효율적으로 길어짐을 확인 후 폐기.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

글로벌 연산 타임라인을 목표(10분) 대비 대폭 축소한 327초로 가속 성공 및 피크 메모리 사용량 170MB 유지. 모델 경량화 달성 후 차세대 생체 모델 적용을 위한 에피유전학 편입 논의.

# [5] 차세대 대식세포 에피유전학(Epigenetics) 스크리닝 마커 통합

## 📌 작업 목표

펩타이드 1차원 서열 정보에 국한하지 않고, 면역 분극화의 실질적 지표인 '염색질 열림 상태(Chromatin Accessibility) 및 히스톤 수식화'를 예측 피처(Feature Space)로 확장.

## 🔧 주요 변경 사항 / 시도한 접근법

- **마커 차원 분리(Dimension Separation) 원칙:** - 가속 페달 역할의 `H3K4me3`와 브레이크 역할의 `H3K27me3`를 '메틸화'로 합치지 않고 각각 별개의 피처 차원으로 철저히 분리.
    
    - 모델이 발현 대기 상태(Bivalent domain) 패턴을 독립적으로 학습할 수 있게 재설계.
        
- **ENCODE 메타데이터 파이프라인 매핑:** B 세포에서 대식세포로 교차 분화된 BLaER1 세포주의 통일된 Replicate Peak(`.bed`) 데이터를 확보하여 초기 MVP용 1/0 이진 인풋 텐서 뼈대 구축.
    
- **설명성 방어 (Explainability):** 결측치가 많고 데이터 설명이 까다로운 `ub`, `ph` 수식화 데이터는 Feature Matrix에서 제외하고 핵심 3축(H3K27ac, H3K4me3, H3K27me3)에 집중.
    

## 💻 핵심 코드 스니펫

Python

```
# 에피유전학적 피처의 생물학적 정반대 역할을 고려한 차원 분할 병합 (MVP 기준)
# shape: (batch_size, sequence_length, feature_dim)
epigenetic_features = torch.cat([
    dna_methylation_tensor,  # 장기 억제 안정성
    h3k27ac_tensor,          # 활성 인핸서 (다이내믹한 즉각 반응)
    h3k4me3_tensor,          # 가속 페달 (독립 차원)
    h3k27me3_tensor          # 브레이크 (독립 차원)
], dim=-1)

# Sequence Embedding과 Epigenetic Features의 Concatenation
multimodal_input = torch.cat([peptide_embedded, epigenetic_features], dim=-1)
```
