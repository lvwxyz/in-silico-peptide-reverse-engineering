# [1] AI 모델 아키텍처 및 하이퍼파라미터 초기 설계

## 📌 작업 목표

염증 유발성(Pro-inflammatory) 모티프를 효과적으로 탐지하고 서열의 전체 맥락을 이해할 수 있는 인실리코(In silico) 모델 아키텍처를 설계하는 것입니다. 짧은 15mer 서열 내에서 발생할 수 있는 정보 소실을 방지하고 최적의 특징 추출(Feature Extraction) 구조를 확립하고자 했습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **1D-CNN + Bi-LSTM 하이브리드 구조 확립**: 국소적 모티프 탐지(CNN) 후 전체 양방향 문맥을 파악(Bi-LSTM)하는 구조 설계
    
- **1D Max Pooling 적용**: Average Pooling 시 강한 염증 신호가 희석되는 문제를 방지하기 위해 Max Pooling(Window=2, Stride=2)을 채택하여 핵심 위험 피처만 보존
    
- **Pre-padding 방식 채택**: 시계열 모델(LSTM)의 특성을 고려하여, 패딩(0)을 서열 앞에 배치해 최신(마지막) 아미노산 정보의 선명도를 유지
    
- **생물학적 인코딩 (BLOSUM62) 적용**: 단순 One-Hot 벡터 대신 아미노산 간 진화적 치환 유사성을 반영한 BLOSUM62 행렬로 임베딩 결정
    

## ❌ 실패하거나 버려진 시도

- **TensorFlow/PyTorch 혼용 시도**: 두 프레임워크 혼용 시 텐서 변환 간 메모리 병목 및 의존성 충돌 우려가 있어 **PyTorch 단일 프레임워크**로 전면 통일.
    
- **Post-padding 시도**: 남는 길이를 뒤에서 0으로 채울 경우, LSTM이 마지막에 무의미한 0을 읽어 실제 아미노산 정보가 희석될(Dilution) 우려가 있어 기각함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

PyTorch 기반의 하이브리드 모델 뼈대와 하이퍼파라미터 초기값(CNN Filter=32, LSTM Hidden=64 등)이 확정되었습니다. 이론적 설계가 완료됨에 따라 이를 학습시킬 실제 IEDB 데이터 수집 및 정제 단계로 이동했습니다.

# [2] IEDB 데이터 전처리 파이프라인 구축

## 📌 작업 목표

IEDB(Immune Epitope Database)에서 추출한 비정형 펩타이드 서열 데이터를 모델이 학습할 수 있는 정형화된 데이터셋으로 정제하는 것입니다. 서열 길이 제약 및 비표준 아미노산을 처리하여 데이터의 품질을 확보하는 것이 핵심이었습니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **길이 필터링**: 모델의 최대 입력 길이에 맞춰 8~15mer 서열만 추출 (`raw_data['length'] >= 8 & <= 15`)
    
- **비표준 데이터 정제**: B, J, O, U, X, Z 등 자연계 표준 아미노산 20종에 속하지 않는 문자가 포함된 서열 제거 로직 구현
    
- **중복 서열 제거**: 동일 서열에 대한 중복 데이터를 통합(`drop_duplicates`)하여 편향 방지 처리
    

## 💻 핵심 코드 스니펫

Python

```
# 비표준 아미노산 필터링 및 길이 제한 적용 로직
standard_aa = set("ACDEFGHIKLMNPQRSTVWY")

# 1. 8~15mer 길이 제한
cleaned_df = raw_data[(raw_data['length'] >= 8) & (raw_data['length'] <= 15)].copy()

# 2. 20종 표준 아미노산만 허용
cleaned_df = cleaned_df[cleaned_df['sequence'].apply(

)]
```

# [3] 데이터 라벨링 매핑 및 BLOSUM62 인코딩 적용

## 📌 작업 목표

수집된 데이터의 정답 라벨(Positive/Negative) 분포를 분석하고, 전처리된 서열을 `PyTorch Dataset` 클래스 내에서 BLOSUM62 기반의 2D 텐서 배열로 변환하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **데이터 분포 분석 및 이진 라벨 통합**: 전체 데이터가 Positive(~54.9%), Negative(~45.1%)로 균형(Balance)을 이룸을 확인. `Positive-Low`, `Positive-High` 등의 파생 라벨을 모두 1(위험)로 통합하고, `Negative`를 0(안전)으로 매핑.
    
- **Pre-padding 기반의 인코딩 로직 구현**: 15mer 미만 서열의 앞부분을 0 행렬로 채우고, 나머지 위치에 BLOSUM62 치환 점수를 할당하는 변환 함수 완성.
    

## 💻 핵심 코드 스니펫

Python

```
def encode_blosum62(self, seq):
    encoding = np.zeros((self.max_len, 20))
    start_pos = self.max_len - len(seq) # Pre-padding 적용 구간
    
    for i, aa in enumerate(seq):
        for j, ref_aa in enumerate(self.aa_order):
            # BLOSUM62 행렬 기반 유사도 점수 매핑
            encoding[start_pos + i, j] = self.matrix[aa][ref_aa]
    return encoding
```

## ❌ 실패하거나 버려진 시도

- **가중치 조정(Class Weight) 및 오버샘플링**: IEDB 특성상 라벨 불균형을 예상하여 손실 함수 가중치(`BCEWithLogitsLoss(pos_weight=...)`)를 세팅하려 했으나, 탐색 결과 분포가 매우 균등(1:1 비율)하여 불필요한 연산 오버헤드로 판단, 표준 BCE Loss로 선회함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

전처리, 라벨링, 인코딩이 모두 포함된 `PeptideDataset` 클래스가 완성되어 모델 학습을 위한 데이터 준비가 100% 완료되었습니다. 다음으로 전체 역설계를 수행할 AI 모델 추론부와 유전 알고리즘(GA) 통합 코드를 작성했습니다.

# [4] AI 예측 엔진 및 유전 알고리즘(GA) 기반 역설계 루프 통합

## 📌 작업 목표

학습된 모델의 추론(Inference) 결과를 기반으로, 생체 적합성(낮은 염증 확률)과 물리화학적 조건(GRAVY, Charge)을 동시에 만족하는 펩타이드 서열을 생성하는 유전 알고리즘(GA) 역설계 파이프라인을 최종 완성하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **하이브리드 AI 모델 클래스 구현**: `Conv1d`와 `LSTM` 연결 시 필요한 차원 전치(`transpose(1, 2)`) 및 양방향 은닉 상태($H_t$) 결합(`torch.cat`) 구현.
    
- **다목적 Cost Function (적합도 함수) 정의**: AI의 염증 확률 예측값($P_{FBR}$)과 BioPython을 이용한 수치(GRAVY, 순전하)를 합산하여 평가. 목표 범위 이탈 시 페널티를 부여하는 로직 구축.
    
- **유전 알고리즘(GA) 연산자 세팅**: 돌연변이(Mutation, 5%) 및 교차(Crossover) 연산을 통해 우수 형질(상위 20%)을 보존하며 최적화 수행.
    

## 💻 핵심 코드 스니펫

Python

```
# 1. AI 모델 특징 추출부 (Tensor Dimension Handling)
def forward(self, x):
    x = x.transpose(1, 2)  # CNN 입력용 변환 (Batch, Channel, Length)
    x = self.pool(self.relu(self.cnn(x)))
    
    x = x.transpose(1, 2)  # LSTM 입력용 복구 (Batch, Length, Channel)
    lstm_out, (hn, cn) = self.lstm(x)
    
    # 정방향, 역방향 은닉 상태(Hidden State) 결합
    h_t = torch.cat((hn[-2,:,:], hn[-1,:,:]), dim=1) 
    return self.sigmoid(self.fc(h_t))

# 2. 역설계 제약조건 Cost 평가 로직
def calculate_cost(seq, model, config, encode_fn):
    # AI 기반 염증 확률 추론
    p_fbr = model(torch.FloatTensor(encode_fn(seq)).unsqueeze(0)).item()
    
    # 목표 물리화학 범위 이탈량(Penalty) 계산
    gravy = ProteinAnalysis(seq).gravy()
    tg = config["constraints"]["gravy_target"]
    penalty_g = 0 if tg[0] <= gravy <= tg[1] else min(abs(gravy-tg[0]), abs(gravy-tg[1]))
    
    # 다목적 최적화 가중치 합산
    total_cost = (w["w_ai"] * p_fbr) + (w["w_gravy"] * penalty_g) + (w["w_charge"] * penalty_c)
    return total_cost, p_fbr, gravy
```
