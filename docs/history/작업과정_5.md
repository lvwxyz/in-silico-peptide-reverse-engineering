# [1] 데이터 파이프라인 구축 및 생물학적 인코딩 전략 수립

## 📌 작업 목표

AI 모델이 펩타이드 서열의 생물학적 의미를 이해할 수 있도록 학습 데이터를 정제하고 최적의 임베딩 방식을 적용하는 데이터 전처리 파이프라인을 구축한다. IEDB 데이터셋의 불균형 문제를 진단하고 해결한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **데이터 소스 및 정제:** IEDB T-cell Assay 데이터에서 최대 길이 15-mer 이하의 선형 펩타이드 서열만 필터링. 면역 반응 결과(Positive-Low, High 등)를 이진 라벨(위험=1, 안전=0)로 통합 매핑.
    
- **생물학적 인코딩 (BLOSUM62):** 단순 One-hot 인코딩 대신, 아미노산 간의 진화적 치환 확률을 반영하는 BLOSUM62 치환 행렬을 적용하여 2D 텐서로 변환.
    
- **Pre-padding 도입:** 시계열 모델(LSTM)이 서열의 마지막(실제 핵심 정보)을 잃지 않고 선명하게 기억하도록, 15-mer 미만 서열의 빈칸을 앞쪽(Pre)에 0으로 채움.
    
- **텐서 사전 변환:** 학습 속도 향상을 위해 DataLoader 내부가 아닌 초기화 단계에서 전체 데이터를 GPU 학습용 텐서 포맷으로 사전 변환(`prepare_data`).

```
# BLOSUM62 기반 Pre-padding 인코딩 로직 적용
def encode_peptide(seq, max_len=15):
    encoding = np.zeros((max_len, 20))
    start_pos = max_len - len(seq) # Pre-padding
    for i, aa in enumerate(seq):
        if aa in amino_acids:
            for j, ref_aa in enumerate(amino_acids):
                encoding[start_pos + i, j] = matrix[aa][ref_aa]
    return encoding
```

## ❌ 실패하거나 버려진 시도

- **클래스 가중치(Class Weighting) 강제 적용:** 초기에는 IEDB 데이터의 라벨 불균형을 예상하여 손실 함수에 가중치(`pos_weight`)를 주거나 언더샘플링을 시도했으나, 실제 데이터 분포가 Positive(~55%) vs Negative(~45%)로 균등함을 확인하고 불필요한 연산 오버헤드를 막기 위해 표준 학습으로 선회함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

데이터 전처리 및 인코딩을 수행하는 `PeptideDataset` 클래스가 완성되어 모델 학습 준비가 완료됨. 이를 바탕으로 특징 추출을 위한 딥러닝 모델(Pipeline A) 아키텍처 설계로 넘어감.

# [2] 하이브리드 AI 엔진(Pipeline A) 설계 및 손실 함수 안정화

## 📌 작업 목표

펩타이드 서열의 국소적 염증 유발 모티프와 서열 전체의 양방향 맥락을 동시에 파악하여 면역 거부반응($P_{FBR}$)을 정확하게 예측하는 딥러닝 아키텍처를 설계한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **1D-CNN + Bi-LSTM 하이브리드 설계:** - `1D-CNN` (Kernel=3, Filters=32): 3~4개 아미노산 단위의 국소적 모티프 1차 탐지.
    
    - `Bi-LSTM` (Hidden=64): CNN이 추출한 특징을 양방향으로 읽어 Global Context ($H_t$) 벡터 생성.
        
- **XAI를 위한 Max Pooling 제거:** 원래 계획했던 1D Max Pooling을 생략하여 서열 길이(15)를 그대로 유지. 이를 통해 후반부 XAI 시각화 시 아미노산 글자와 모델의 Attention이 1:1로 매칭되도록 설계.
    
- **학습 안정화:** 수치적 불안정성(Cascading Errors)을 잡기 위해 출력층을 `BCELoss`에서 로짓(Logit)을 직접 다루는 `BCEWithLogitsLoss`로 전면 교체.
    

## 💻 핵심 코드 스니펫

Python

```
class HybridPeptideModel(nn.Module):
    def __init__(self, input_dim=20, cnn_filters=32, lstm_hidden=64):
        super(HybridPeptideModel, self).__init__()
        # 1D-CNN: 국소 모티프 추출
        self.cnn = nn.Conv1d(input_dim, cnn_filters, kernel_size=3, padding=1)
        self.relu = nn.ReLU()
        # XAI 매핑을 위해 MaxPool 생략, 바로 Bi-LSTM으로 전달
        self.lstm = nn.LSTM(cnn_filters, lstm_hidden, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(lstm_hidden * 2, 1) # 양방향(x2) -> 1(확률)

    def forward(self, x):
        x = self.relu(self.cnn(x.transpose(1, 2))).transpose(1, 2)
        lstm_out, (hn, cn) = self.lstm(x)
        h_t = torch.cat((hn[-2,:,:], hn[-1,:,:]), dim=1) # 정방향/역방향 은닉 상태 병합
        return self.fc(h_t) # BCEWithLogitsLoss를 위해 Sigmoid 생략
```

## ❌ 실패하거나 버려진 시도

- **단일 1D-CNN 아키텍처:** CNN 구조만 사용했을 때, 서열 양 끝단이 접히며 결합하는 등 거리가 먼 아미노산 간의 상호작용을 인지하지 못해 분류 정확도가 0.6 초반대에서 정체됨. 이를 해결하기 위해 Bi-LSTM을 결합하는 하이브리드 방식으로 구조를 변경함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

AUROC 및 F1-Score가 안정적으로 상승하며 신뢰할 수 있는 염증 예측 엔진 구축 완료. 이 모델을 '적합도 평가 심사위원'으로 활용하여 실제 펩타이드를 설계하는 역설계 최적화 알고리즘 개발로 이어짐.

# [3] 유전 알고리즘(GA) 기반 다목적 역설계 루프(Pipeline B) 구현

## 📌 작업 목표

학습된 AI 모델을 바탕으로, 염증 확률을 낮출 뿐만 아니라 수용성, 전하량, 상동성 등 물리화학적/생물학적 제약 조건을 동시에 만족하는 최적의 펩타이드를 자동 생성(역설계)한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **다목적 비용 함수(Cost Function) 설계:** - $P_{FBR}$ (AI 예측치), GRAVY (소수성 페널티), $Q_{net}$ (순전하 페널티), $H_{CD47}$ (기능 유지를 위한 상동성 보상), 절대 제약 조건(길이 위반 시 +999)을 통합.
    
    - 베이스 소재의 밀도(Ligand density)에 비례하여 GRAVY 페널티 가중치를 동적으로 스케일링하는 로직 추가.
        
- **Warm-start (AUTO 모드) 초기화:** 완전 무작위 서열에서 진화를 시작하면 수렴 시간이 과도하게 소요됨. hCD47 핵심 서열(`EVTELTREGETYERY`)을 초기 인구의 80%에 베이스로 삽입하여 탐색 효율 극대화.
    
- **벡터 연산 최적화:** 50개 유전자 개체군의 비용 함수 연산 시 파이썬의 중첩 `for` 루프 병목을 막기 위해 텐서 배열 형태의 일괄 연산(Batch Processing) 처리 적용 (목표 10분 이내 달성).
    

## 💻 핵심 코드 스니펫

Python

```
def calculate_cost(seq, model, config):
    # 1. AI FBR 확률 추론
    p_fbr = infer_ai_model(seq, model)
    
    # 2. 물리화학적 특성(GRAVY, Charge) 페널티 계산 (제약 범위 이탈 시 부과)
    gravy, charge = get_physicochemical_props(seq)
    density_scale = 1.0 + (config["ligand_density"] / 100.0) # 밀도 연동 스케일링
    penalty_g = max(0, abs(gravy) - target_g) * density_scale
    
    # 3. CD47 기능 유지 상동성(Homology) 보상 점수
    h_cd47 = calculate_homology(seq, ref_seq="EVTELTREGETYERY")
    
    # 최종 다목적 수식 계산
    w = config["weights"]
    cost = (w["ai"] * p_fbr) + (w["gravy"] * penalty_g) + (w["charge"] * penalty_c) - (w["cd47"] * h_cd47)
    return cost
```

## ❌ 실패하거나 버려진 시도

- **조기 수렴 (Local Optimum) 문제:** 초기 세대 평가 시, 상위 1등 개체에게만 교배 기회를 몰아주어(선택압 과다) 10세대 만에 50개 펩타이드 서열이 획일화됨.
    
- **해결책:** 파레토 프론트(Pareto Front) 기반 평가 및 토너먼트 선택압 완화를 도입하고 '적응형 돌연변이(Adaptive Mutation)' 로직을 추가하여 유전적 다양성을 확보함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

최적 서열 도출 최적화 루프가 성공적으로 작동함. 이후 서열 길이에 따른 대응과 공간 구조 데이터를 반영하기 위한 시스템 고도화 단계로 돌입함.

# [4] 3D 구조 데이터(SASA) 통합 및 Smart Router 모듈 적용

## 📌 작업 목표

단순 1D 서열 정보의 한계를 넘어 3D 입체 구조 정보를 학습에 결합(Multi-modal)하고, 입력되는 펩타이드 길이에 따라 최적의 모델로 분기하는 라우팅 시스템을 도입한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **3D SASA (Solvent Accessible Surface Area) 피처 통합:** - `ColabFold(AlphaFold2)`를 활용해 PDB 구조를 예측한 뒤 `Biopython`으로 잔기별 SASA 값을 추출.
    
    - SASA 값 스케일(150 이상)이 인코딩 수치를 압도하여 가중치가 붕괴되는 것을 막기 위해 값을 200으로 나누어 정규화(Normalization).
        
    - 1D Sequence 텐서와 3D SASA 텐서를 결합하는 Early Fusion 엔진 구축.
        
- **Smart Router (동적 분기) 엔진 도입:** - 긴 서열과 짧은 서열 처리에 대한 효율성 트레이드오프 해결.
    
    - 서열 길이 $\le$ 15: 연산이 가벼운 `CNN Only 엔진`으로 라우팅.
        
    - 서열 길이 $>$ 15: 깊은 문맥 파악이 필요한 `Hybrid (CNN+LSTM) 엔진`으로 라우팅.
        

## 💻 핵심 코드 스니펫

Python

```
# Smart Router 로직 
def smart_router_predict(seq, cnn_model, hybrid_model):
    seq_length = len(seq)
    
    if seq_length <= 15:
        # 속도 및 국소 패턴 위주의 짧은 서열 처리
        return cnn_model(encode_peptide(seq))
    else:
        # 글로벌 문맥 파악이 필수적인 긴 서열 처리
        return hybrid_model(encode_peptide(seq))
```

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

1D 서열과 3D 구조 데이터가 통합된 멀티모달 파이프라인이 완성되었으며, Smart Router를 통해 추론 효율과 모델 정확도(AUROC) 간의 밸런스를 모두 확보하는 최종 In Silico Pipeline 개발을 완료함.
