# [대화 01] 멀티오믹스 기반에서 물리화학적 특성 기반 펩타이드 역설계로의 방향 선회

## 📌 작업 목표

초기 공간 다중 후성유전체(Spatial Multi-Epigenomics) 데이터를 활용하려던 기획에서 벗어나, 대식세포의 면역 회피 생체재료를 설계하기 위한 현실적이고 안정적인 데이터 소스 및 모델 입력(Input) 체계를 확정합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **입력 데이터 변경:** IUPAC 표기 서열과 후성유전 마커 결합 방식에서, 펩타이드 서열(Sequence) 자체와 물리화학적 지표(GRAVY, Charge 등)를 조합하는 하이브리드 입력 체계로 전면 개편
    
- **목표 지표(KPI) 재설정:** 염증 확률(P_FBR) 최소화, 소수성 지수(GRAVY) 0 수렴, CD47 핵심 서열 보존으로 역설계 탐색 기준 명확화
    

## ❌ 실패하거나 버려진 시도

- **후성유전 데이터 사용 포기:** DNA 메틸화, 히스톤 아세틸화 등의 마커를 독립적인 입력 벡터로 구성하려 했으나, 퍼블릭 데이터의 잦은 결측치와 품질 문제로 인해 모델 학습 불가 판정. '생물학적 설명력'보다 '데이터 기반 구현 가능성'을 우선하여 폐기함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

IEDB(Immune Epitope Database)를 메인 데이터셋으로 채택하고, 서열 기반 1D-CNN + Bi-LSTM 멀티모달 딥러닝과 유전 알고리즘(GA)을 결합하는 파이프라인으로 최종 기술 스택과 노선을 확정했습니다.

# [대화 02] 데이터 전처리 파이프라인 및 진화적 임베딩 구축

## 📌 작업 목표

AI가 단순 알파벳 텍스트(단백질 서열)를 생물학적 의미가 담긴 수치로 이해할 수 있도록 데이터를 정제하고 학습용 2D 텐서로 인코딩하는 데이터셋 클래스를 완성합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **BLOSUM62 기반 임베딩:** 아미노산 간 진화적 치환 가능성을 반영한 가중치 행렬을 사용하여 서열을 임베딩함
    
- **데이터 증강 및 필터링:** IEDB 데이터 중 최대 길이(max_len) 15-mer 이하 서열만 필터링하고 Pre-padding 기법 적용
    
- **이진 라벨링 통합:** Positive-Low, Positive-High 등의 파생 라벨을 1(위험)로 통합, Negative를 0(안전)으로 매핑
    

## ❌ 실패하거나 버려진 시도

- **가중치 손실 함수(Class Weight) 및 오버샘플링 적용 취소:** IEDB 데이터의 클래스 불균형을 예상해 `BCEWithLogitsLoss(pos_weight)`를 설정하려 했으나, 실제 데이터 분포가 약 1:1로 균등함을 확인하여 연산 오버헤드 방지 차원에서 표준 BCE Loss로 선회함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

라벨링, 임베딩, 텐서 변환이 통합된 `PeptideDataset` 클래스를 구축하여 100% 데이터 준비를 완료하였고, 본격적인 예측 모델(Predictor) 아키텍처 설계 단계로 진입했습니다.

# [대화 03] 하이브리드 예측 모델(Pipeline A) 아키텍처 설계 및 최적화

## 📌 작업 목표

단백질 서열의 국소적 독성 모티프와 전역적 문맥(3차원 폴딩 고려)을 동시에 파악하여 높은 신뢰도로 면역 반응을 예측하는 AI 아키텍처를 구축합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **1D-CNN + Bi-LSTM 직렬 결합 (SeqHybrid):** 3-mer 커널의 CNN으로 핵심 모티프를 추출하고, Bi-LSTM으로 양방향 맥락을 파악하는 구조 확립
    
- **안정화 기법 적용:** 모델의 과적합(Overfitting) 방지를 위해 Dropout 레이어 및 L2 정규화(Weight Decay) 적용
    

## ❌ 실패하거나 버려진 시도

- **순수 CNN / RNN 단독 구조의 한계:** CNN만 차용했을 시 거리가 먼 아미노산 상호작용 인지 실패로 정확도가 0.6 초반대에서 정체됨. 깊은 층 구성 시 발생한 역전파 소실(Cascading Error)을 잔차 연결(Residual Connection)과 어텐션 기법을 도입하여 방어함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

검증 데이터 기준 AUROC 0.7235를 달성하여 시스템의 목표 허들(>0.7)을 돌파했습니다. 학습된 모델을 유전 알고리즘 내에서 펩타이드 적합성을 평가하는 '심사위원 모델'로 확정했습니다.

# [대화 04] 다목적 유전 알고리즘(Pipeline B) 고도화 및 연산 최적화

## 📌 작업 목표

AI 예측 결과를 바탕으로 체내 수용성, 전하량, 면역 회피 조건을 모두 만족시키는 최적의 펩타이드 서열을 생성(역설계)하며, 연산 병목을 제거하여 10분 이내에 탐색을 완료합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **Warm-start 초기화:** 수렴 속도 단축을 위해 hCD47 핵심 서열(EVTELTREGETYERY)을 초기 0세대 인구의 80%에 강제 주입
    
- **벡터 연산 최적화:** 50개 개체군 평가 시 발생하는 중첩 `for` 루프 병목을 텐서(Tensor) 배열 일괄 연산(Batch Processing)으로 개편
    

## ❌ 실패하거나 버려진 시도

- **엘리트주의 교배로 인한 조기 수렴(Local Optimum):** 상위 1등 개체에게만 교배를 집중시켰더니 10세대 만에 서열이 획일화됨. 파레토 프론트(Pareto Front) 기반 평가 및 '적응형 돌연변이(Adaptive Mutation)'를 도입하여 유전적 다양성 문제를 해결함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

연산 튜닝을 통해 50세대 최적화 루프를 327초(약 5.5분)만에 돌파하며 하드웨어 효율성(피크 메모리 170MB)을 달성했습니다. 이후 모델 결합의 당위성을 입증하기 위한 Ablation Study 실험으로 넘어갔습니다.

# [대화 05] Ablation Study 및 K-Fold 교차 검증: 아키텍처 재평가와 경량화

## 📌 작업 목표

1D-CNN과 Bi-LSTM 결합의 기술적 정당성을 증명하기 위해, 각 단독 모델과의 공정한 절제 연구(Ablation Study) 및 5-Fold 교차 검증(CV)을 수행하여 객관적 지표(AUROC, AUPRC, FP)를 추출합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **공정 파라미터 보정:** CNN 단독 모델의 수용 용량(Capacity)을 하이브리드와 맞추기 위해 CNN 필터 수를 64로 확장
    
- **평가 지표 다각화:** 50 Epoch, 5-Fold CV 환경에서 정밀도-재현율 곡선 면적(AUPRC) 및 위양성(False Positive) 건수 트래킹 로직 추가
    

## 💻 핵심 코드 스니펫

Python

```
# K-Fold 평가 루프 내 객관적 지표(위양성 및 AUPRC) 산출 로직
preds_flat = np.array(all_preds).flatten()
labels_flat = np.array(all_labels).flatten()

# [지표 1] AUROC
auroc = roc_auc_score(labels_flat, preds_flat)
# [지표 2] AUPRC (불균형 데이터 정밀도 평가)
auprc = average_precision_score(labels_flat, preds_flat)
# [지표 3] 혼동 행렬에서 오진(FP) 추출
binary_preds = (preds_flat > 0.5).astype(int)
tn, fp, fn, tp = confusion_matrix(labels_flat, binary_preds).ravel()
```

## ❌ 실패하거나 버려진 시도

- **하이브리드 구조의 예상외 패배 (오버피팅):** 5-Fold CV 결과, 1D-CNN 단독(AUROC 0.8423)이 하이브리드(0.8230)를 크게 압도함. 최대 길이 15 아미노산이라는 '짧은 서열' 환경에서는 1D-CNN의 수용 영역(Receptive Field)만으로도 전체 문맥 파악이 완벽히 끝나며, 불필요한 LSTM 결합은 위양성(FP)을 오히려 증가시킨다는 사실을 발견하고 하이브리드 고집을 버림.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

초기 가설을 기각하고 1D-CNN 단독 아키텍처로 모델을 경량화했습니다. 이로 인해 임상 활용 최소 기준인 AUROC 0.84를 달성하며 연산 효율성 및 위양성(FP) 방어를 극대화했습니다.

# [대화 06] 길이 적응형(Length-Adaptive) 스마트 라우터 설계 및 길이 페널티 도입

## 📌 작업 목표

생체재료의 타겟 스펙을 '하이드로젤 부착 모티프'에서 '독립적 폴리펩타이드(길이 30 이상)'로 상향 조정함에 따라, 서열 길이에 따라 최적의 AI 모델을 선택하는 동적 분기 아키텍처를 도입하고 과도한 서열 증가를 방지합니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **동적 라우팅(Smart Router) 체계 수립:** - `length <= 15`: 1D-CNN Only (빠른 연산 및 오버피팅 방지)
    
    - `length > 15`: 1D-CNN + Bi-LSTM Hybrid (긴 서열의 3차원적 장거리 문맥 파악)
        
- **비용 함수(Cost Function) 개선:** 유전 알고리즘에서 서열이 무한정 길어져 합성 비용이 증가하는 것을 막기 위한 페널티(Length Penalty) 부여
    

## 💻 핵심 코드 스니펫

Python

```
def calculate_cost(seq, model, aa_to_idx, config):
    # ... 기존 지표 계산(p_fbr, gravy, charge, h_score) ...
    
    # 💡 서열 길이에 따른 최적화 페널티(Length Penalty) 부여
    optimal_length = 18 
    length_penalty = 0.0
    if len(seq) > optimal_length:
        length_penalty = (len(seq) - optimal_length) * 0.1 # 초과 1글자당 0.1 벌점
    
    cost = (w["ai"] * p_fbr) + (w["gravy"] * abs(gravy)) + (w["charge"] * abs(charge)) - (w["cd47"] * h_score) + length_penalty
    
    return cost, p_fbr, h_score, gravy, charge
```

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

Ablation Study에서 패배하여 버려질 뻔한 하이브리드 모델을 '긴 서열 전용 정밀 예측기'로 재활용하는 완벽한 공학적 논리를 수립했습니다. 데이터 특성(길이)에 맞춰 최적의 모델 엔진을 자동 할당하는 Length-Adaptive AI 파이프라인(최종안)을 성공적으로 확정했습니다.
