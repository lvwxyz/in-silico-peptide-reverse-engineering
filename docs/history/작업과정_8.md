# 1. 초기 데이터 파이프라인 및 베이스라인 모델 구축

## 📌 작업 목표

아미노산 서열 데이터를 모델이 이해할 수 있는 벡터 형태로 변환하고, 펩타이드의 면역 반응(독성) 여부를 판단하기 위한 초기 분류 모델(Baseline) 아키텍처를 설계하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- `BLOSUM62` 치환 행렬을 이용해 생물학적 서열 특성을 숫자 텐서로 임베딩하는 데이터 전처리 파이프라인 구축
    
- `ESM-2` 사전 학습 모델을 활용한 토크나이저 결합(이중 임베딩 시도)
    
- 서열의 독성 부위를 추출하는 **1D-CNN**과 문맥을 파악하는 **Bi-LSTM** 단일 모델 설계
    
- 두 신경망을 결합한 하이브리드 아키텍처 2종(직렬 연결형 `SeqHybrid`, 병렬 연결형 `ParallelHybrid`) 구현
    

## 💻 핵심 코드 스니펫

Python

```
# 직렬 형식의 초기 hybrid model (CNN -> Max-Pooling -> Bi-LSTM)
class SeqHybrid(nn.Module):
    def __init__(self):
        super().__init__()
        self.cnn = nn.Conv1d(20, 64, kernel_size=3, padding=1)
        self.lstm = nn.LSTM(64, 64, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(128, 1)
    def forward(self, blosum, ids, mask):
        x = F.relu(self.cnn(blosum.permute(0, 2, 1)))
        out, _ = self.lstm(F.max_pool1d(x, 2).permute(0, 2, 1))
        return torch.sigmoid(self.fc(out[:, -1, :]))
```

## ❌ 실패하거나 버려진 시도

- 1D-CNN 단독 모델 및 병렬(Parallel) 하이브리드 모델은 피처 결합 과정의 비효율성으로 성능이 저조하여 메인 후보군에서 배제함.
    
- ESM-2를 보조(Auxiliary)로 결합하려 했으나, 오히려 역효과(오버피팅 및 노이즈)를 내는 것으로 확인됨.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

직렬 방식의 `SeqHybrid`가 가장 합리적인 구조로 채택되었으나, 단순 `Bi-LSTM` 모델보다 성능이 떨어지는 이상 현상이 발견됨. 원인 분석 결과, CNN에서 추출된 피처가 LSTM으로 넘어가는 과정에서 정보가 유실되는 'Cascading Errors'로 판명되어 모델 구조 개선으로 넘어감.

# 2. Cascading Errors 해결 및 하이브리드 모델 고도화

## 📌 작업 목표

초기 직렬 하이브리드 모델의 정보 유실(Cascading Error) 문제를 해결하고, 서열 내 중요한 아미노산 위치를 모델이 스스로 파악할 수 있도록 네트워크 구조를 고도화하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **Residual Connection(잔차 연결) 도입**: CNN 요약본뿐만 아니라 원본 BLOSUM 데이터를 Concat하여 지름길(Shortcut) 형태로 LSTM에 전달 (`SeqHybridResidual`)
    
- **Attention Mechanism(어텐션) 적용**: 각 아미노산 위치에 형광펜을 칠하듯 가중치를 학습하여 문맥 벡터를 하나로 응축하는 어텐션 층 추가 (`SeqHybridAttention`)
    

## 💻 핵심 코드 스니펫

Python

```
# 1. 잔차 연결(Residual Connection) 적용
def forward(self, blosum, ids, mask):
    x_cnn = F.relu(self.cnn(blosum.permute(0, 2, 1))) 
    # 지름길: CNN 요약본과 원본 블로섬 데이터를 결합 (Concat)
    x_combined = torch.cat((blosum.permute(0, 2, 1), x_cnn), dim=1) 
    out, _ = self.lstm(x_combined.permute(0, 2, 1))
    return torch.sigmoid(self.fc(out[:, -1, :]))

# 2. 어텐션 매커니즘(Attention) 적용
def forward(self, blosum, ids, mask):
    x = F.relu(self.cnn(blosum.permute(0, 2, 1)))
    out, _ = self.lstm(x.permute(0, 2, 1))
    attn_weights = F.softmax(self.attention(out), dim=1) # 어텐션 스코어 계산
    context_vector = torch.sum(attn_weights * out, dim=1) # Weighted Sum
    return torch.sigmoid(self.fc(context_vector))
```

## ❌ 실패하거나 버려진 시도

- Max-Pooling 과정에서의 정보 손실을 막기 위해 단순 풀링 제거 모델(`No Pool`)을 테스트했으나, 시퀀스 길이가 유지되어 연산량만 늘고 유의미한 성능 향상은 없었음.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

구조적 결함을 수정한 Residual 및 Attention 모델이 성공적으로 구현됨. 이제 이 다양한 모델들(총 10개) 중 어떤 모델이 in silico 환경에서 가장 적합한지 객관적으로 평가하기 위한 대규모 벤치마킹 및 다목적 최적화 기준으로 넘어감.

# 3. 데이터 스케일업 및 다목적 모델 평가(Ablation Study) 기준 확립

## 📌 작업 목표

학습 데이터 규모를 대폭 늘려 모델을 강건하게 학습시키고, 단순히 정확도(AUROC)만 높은 모델이 아닌 연산 비용(메모리, 시간)까지 고려한 '최적의 AI 심사위원'을 선발하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **샘플 사이즈 3배 확장**: 20,000개에서 60,000개로 데이터 증강 적용.
    
- 멀티-런(3-Run) 통계 수합을 통한 모델 안정성(Std) 검증 추가.
    
- `Total_Score` 도입: AUROC 성능을 기본으로 하되, In Silico 역설계 환경 특성상 치명적인 '메모리 사용량'과 '학습 소요 시간', '성능 편차'를 감점 요소(Penalty)로 적용한 평가 수식 설계.
    

## 💻 핵심 코드 스니펫

Python

```
# 통합 점수(Total Score) 다목적 최적화 산출
W_STD = 0.01
W_TIME = 0.001
W_MEM = 0.5  # In silico 특성 반영 (메모리 제약 가중치 큼)

df_report['Total_Score'] = (
    df_report['Mean_AUROC']
    - (W_STD * df_report['Std_AUROC'])
    - (W_TIME * df_report['Mean_Time'])
    - (W_MEM * df_report['Peak_Mem_MB'])
)
df_sorted = df_report.sort_values(by='Total_Score', ascending=False)
```

## ❌ 실패하거나 버려진 시도

- `ESM-2 Only` 및 `Main-Aux Hybrid` 모델은 1회 Run에 최대 1,680초(28분 이상)가 소요되고 과도한 GPU/RAM 자원을 점유하는 것이 확인되어, 시간 관계상 트래킹 딕셔너리에서 주석 처리하고 최종 후보군에서 영구 제외함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

데이터 확장 후 발생하던 메모리 병목 문제를 해결했으며, 통합 점수(`Total_Score`)를 기준으로 역설계를 주도할 가장 가볍고 정확한 최적의 모델(가중치) 로딩에 성공함. 선발된 모델을 평가 엔진으로 사용하는 유전 알고리즘 파이프라인 개발로 넘어감.

# 4. 역설계 유전 알고리즘(GA) 도입 및 생태계 조기 수렴 버그 해결

## 📌 작업 목표

선정된 AI 모델을 심사위원으로 삼아, 무작위 펩타이드 풀에서 안전한(면역 거부 없는) 서열을 교배·변이시켜 최적 서열을 '역설계'하되, 탐색이 멈춰버리는 현상(조기 수렴)을 해결하는 것입니다.

## 🔧 주요 변경 사항 / 시도한 접근법

- Cost Function에 AI 평가 점수(`p_fbr`), GRAVY, Charge, hCD47 모티프 보너스 가중치 부여 로직 구현.
    
- **적응형 돌연변이(Adaptive Mutation Rate) 알고리즘** 개발: 유전자 생태계 다양성(Unique sequence ratio)을 실시간 추적하여, 풀의 획일화가 감지되면 시스템이 개입해 변이율을 폭증시키는 로직 추가.
    

## 💻 핵심 코드 스니펫

Python

```
# [적응형 돌연변이 로직] 다양성 붕괴 방어
unique_seqs = len(set([x[5] for x in scored_pop]))
current_diversity = (unique_seqs / pop_size) * 100

if current_diversity < 15.0:
    # 위기 상황: 다양성이 15% 미만이면 돌연변이율을 최대 50%까지 동적 상승
    current_mutation_rate = min(0.50, current_mutation_rate + 0.15) 
    status = "⚠️ 위기 (돌연변이율 상승↑)"
else:
    # 안정 상황: 기본 변이율 10% 유지
    current_mutation_rate = base_mutation_rate 
    status = "✅ 정상"
```

## ❌ 실패하거나 버려진 시도

- 초기 구현 시 고정된 돌연변이율(10%)을 사용했으나, 특정 서열 형태에 개체군이 갇혀버리며(Local Minima) 이후 세대에서 면역 회피율 최적화가 더 이상 진행되지 않는 조기 수렴(Premature Convergence) 현상이 발생하여 폐기.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

적응형 돌연변이 도입 후 진화 그래프에서 다양성 반등 및 P_FBR 스코어의 지속적 하락(임상 기준선 40% 이하 돌파)이 검증됨. 알고리즘의 신뢰성이 입증되었으므로, 가중치 변화에 따른 최종 **민감도 분석(Sensitivity Analysis)** 단계로 이관하여 파이프라인 설계를 마무리함.
