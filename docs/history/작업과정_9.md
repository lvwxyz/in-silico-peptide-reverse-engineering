# [대화 01] 데이터 준비 및 경량화 전략 수립 (Data Reduction & Preprocessing)

## 📌 작업 목표

기존 작업 파일을 분실한 상태에서 데이터 파이프라인의 기초를 신속하게 재구축한다. 메모리 낭비를 방지하고 AI 엔진이 불필요한 노이즈(Noise)까지 학습하는 리스크를 차단하기 위해 6만 개 이상의 원본 데이터를 다이어트하고 생체 적합성 기준 데이터를 정제하는 것을 목표로 한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **긴 단백질 서열 슬라이딩 윈도우 가공:** 알부민(Albumin)이나 IgG1 같은 긴 천연 단백질 서열을 AI가 읽을 수 있는 크기인 15-mer 단위로 쪼개기 위해 한 칸씩 밀어가며 조각을 추출하는 Sliding Window 기법 도입.
    
- **데이터 순도 정제:** 추출된 조각 서열 중 중복되는 시퀀스를 완전히 제거하여 데이터 정밀도 향상.
    
- **기준 라벨링 규칙 확립:** 인체 유래 단백질 서열의 경우 면역 거부 반응이 없는 안전한 상태이므로 무조건 $Target: 0$(안전)으로 통제 라벨링 설정.
    
- **IEDB 에피토프 데이터 수집 파이프라인 기획:** IEDB(Immune Epitope Database)의 데이터셋을 연동하여 면역 반응 수치를 이진 분류(Positive=1, Negative=0)로 변환하는 구조 준비.
    

## 💻 핵심 코드 스니펫 (있는 경우)

Python

```
import pandas as pd

def preprocess_fasta_to_sliding_windows(fasta_sequences, window_size=15):
    """긴 천연 서열을 15-mer 단위로 분할 및 안전군(0) 라벨링 수행"""
    extracted_chunks = []
    for seq in fasta_sequences:
        for i in range(len(seq) - window_size + 1):
            sub_chunk = seq[i:i+window_size]
            extracted_chunks.append({"sequence": sub_chunk, "target": 0})
            
    # 중복 제거 및 데이터프레임 빌드
    df_slim = pd.DataFrame(extracted_chunks).drop_duplicates().reset_index(drop=True)
    return df_slim
```

# [대화 03] 데이터 불균형 극복을 위한 동적 가중치 학습 루프 구축

## 📌 작업 목표

마스터 데이터셋 내에 존재하는 극단적인 클래스 불균형(염증 데이터 92% vs 안전 데이터 8%) 환경을 극복할 딥러닝 손실 함수를 설계한다. 인공지능이 데이터가 많은 한쪽 클래스로만 맹목적으로 찍어서 과적합(Overfitting)을 일으키는 붕괴 상태를 원천 차단하는 것을 목표로 한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **동적 가중치 손실 함수 도입:** 단순 오버샘플링/언더샘플링의 물리적 한계를 넘기 위해 `weight_for_0`와 `weight_for_1`을 데이터 비율의 역수로 계산하여 AI가 소수 데이터를 틀렸을 때 훨씬 강력한 벌점을 부여하도록 설계.
    
- **수치적으로 안정한 손실 함수 교체:** 연산 도중 발생하는 정보 유실 및 Cascading Errors를 제어하기 위해 기존 `BCELoss`를 로짓(Logit) 스케일을 직접 다루는 `BCEWithLogitsLoss`로 전면 교체.
    
- **핵심 평가 지표의 다각화:** 불균형 데이터 환경에서 허황된 수치를 내뱉는 단순 정확도(Accuracy) 모니터링을 지양하고, 진짜 성능을 대변하는 `AUROC` 및 `F1-Score` 중심의 평가 시스템 구축.
    
- **3D 입체 정보 피처 정규화:** Biopython으로 추출한 SASA(원자 노출도) 값(~150)의 수치적 스케일이 아미노산 인코딩 값을 압도하여 학습이 꼬이는 현상을 방지하고자 200으로 나누어 [0, 1] 범위로 조절하는 SASA 정규화 기법 결합.

# [대화 04] SASA 대리 모델(Surrogate AI) 연산 가속화 및 다목적 유전 알고리즘 설계

## 📌 작업 목표

유전 알고리즘(GA) 기반 역설계 탐색 루프 가동 시 수만 번 호출되는 AlphaFold2 구조 계산의 살인적인 연산 오버헤드(시간 병목)를 극복한다. 면역 안전성과 다양한 물리화학적 조건(용해도, 전하량)을 종합 계산하는 다중 목표 비용 함수를 완벽하게 구현하는 것을 목표로 한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **SASA 예측 대리 모델(Surrogate AI) 탑재:** 매 세대마다 AlphaFold를 직접 돌리지 않고도 입체 구조 표면 노출도를 0.001초 만에 유추하는 가벼운 보조 AI(회귀 분석 Surrogate 모델)를 구축하여 파이프라인 결합.
    
- **Warm-start 초기화 기법 주입:** 무작위 서열에서 무지성으로 탐색을 시작할 경우 수렴에 걸리는 막대한 연산 비용을 차단하기 위해, 인체 유래 대식세포 회피 신호 핵심 모티프(`EVTELT`)를 초기 0세대 생성 시점에 고정 강제 주입.
    
- **다중 목표 비용 함수 수식화:** 인공지능이 예측한 염증 확률($P_{FBR}$), 친수성/소수성 밸런스($|GRAVY|$), 생리적 중성 상태를 유도하기 위한 순전하량($|Charge|$), hCD47 상동성 보상 항을 선형 가중 결합한 하이브리드 Cost 함수 설계.
    
- **공학적 시스템 지표 프로파일러 연동:** `time` 및 `psutil` 라이브러리를 GA 루프 전후에 바인딩하여 소요 시간, RAM/VRAM 점유율, 모델 판단 확신도(Certainty)를 실시간 추적하도록 조치.
    

## 💻 핵심 코드 스니펫 (있는 경우)

Python

```
def calculate_multitarget_cost(sequence, p_fbr, gravy, charge, homology_score, config):
    """생물학적 위험도와 물리화학적 제약 조건을 동시에 연산하는 비용 함수"""
    w = config["weights"] # {"p_fbr": 0.5, "gravy": 0.15, "charge": 0.15, "homology": 0.2}
    
    # 제약 조건 벌점 처리 (안전 범위를 벗어날 경우 패널티 부여)
    penalty_gravy = abs(gravy) if gravy > 0 else 0.0 # GRAVY < 0 타겟
    penalty_charge = abs(charge) if not (-0.5 <= charge <= 0.5) else abs(charge)
    
    # 선형 가중 결합 및 CD47 보상 항 차감
    total_cost = (w["p_fbr"] * p_fbr) + (w["gravy"] * penalty_gravy) + \
                 (w["charge"] * penalty_charge) - (w["homology"] * homology_score)
    return total_cost
```

## ❌ 실패하거나 버려진 시도 (있는 경우)

- **고정 돌연변이율 기반 엘리트주의 GA 포기:** 초기 탐색 시 상위 1등 개체 위주로만 자손을 번식시켰더니 10세대 만에 개체군 전체가 특정 서열 형태로 획일화되며 갇혀버리는 조기 수렴(Local Minimum) 결함이 발견됨. 이를 해결하기 위해 파레토 프론트(Pareto Front) 개념과 유전자 생태계 다양성을 모니터링하여 변이율을 폭증시키는 적응형 돌연변이(Adaptive Mutation) 로직으로 대체함.

# [대화 05] 절제 연구(Ablation Study) 기동 및 단독 모델 성능 역전 피벗

## 📌 작업 목표

심사 및 디펜스 과정에서의 기술적 정당성을 확보하기 위해 1D-CNN 레이어와 Bi-LSTM 레이어의 결합 근거를 실험 데이터로 증명한다. 각 신경망 요소의 기여도를 분해(Contribution Decomposition)하고 5-Fold 교차 검증 환경에서 객관적인 성능 지표를 도출하는 것을 목표로 한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **독립 하위 모델 벤치마킹 환경 구축:** 복잡하게 얽혀있는 직렬 하이브리드 모델에서 레이어를 단순히 빼면 입출력 차원이 깨지므로, 'CNN 단독 모델', 'LSTM 단독 모델', '하이브리드 결합 모델' 총 3종의 독립 자동차를 구축해 경쟁시키는 정석적 Ablation 수립.
    
- **최고 점수 추적(Best Score Tracking) 로직 보정:** 특정 에포크에서 하이브리드 모델이 오버피팅되어 최종 성능이 억울하게 감점되는 현상을 방지하기 위해, 학습 루프 도는 동안 가장 높았던 최고 성능(`Best AUROC`)을 트래킹하여 반환하도록 함수 수술.
    

## 💻 핵심 코드 스니펫 (있는 경우)

Python

```
def train_and_track_best_ablation(model, train_loader, val_loader, optimizer, criterion, epochs=30):
    """Ablation Study 환경에서 각 모델의 최고 AUROC 성과를 정확히 포착하는 루프"""
    best_auroc = 0.0
    for epoch in range(epochs):
        model.train()
        for batch_x, batch_y in train_loader:
            optimizer.zero_grad()
            loss = criterion(model(batch_x), batch_y)
            loss.backward()
            optimizer.step()
            
        # 검증 단계 및 최고 점수 트래킹 (공정한 성능 캡처)
        current_val_auroc = evaluate_model_auroc(model, val_loader)
        if current_val_auroc > best_auroc:
            best_auroc = current_val_auroc
            
    return best_auroc
```

## ❌ 실패하거나 버려진 시도 (있는 경우)

- **하이브리드(CNN+LSTM) 모델 아키텍처 중심 기조 기각:** 5-Fold 교차 검증의 뚜껑을 열어본 결과 1D-CNN 단독 모델이 `AUROC 0.8423`을 기록하여 하이브리드 모델(`0.8230`)을 크게 앞지르는 예상 밖의 실험 데이터 직면. 타겟팅하는 펩타이드 서열의 최대 길이가 15 아미노산으로 매우 짧기 때문에 LSTM이 파악해야 할 전역적 장거리 문맥의 실효성이 희석되며, 불필요한 가중치 과적합만 야기한다는 도메인 결론에 도달하여 하이브리드 만능론을 과감히 폐기함.
    

## ✅ 이 단계의 결과 / 다음 단계로 넘어간 이유

초기 가설을 기각하고 짧은 서열 구간에 한해 가장 가볍고 정확한 '1D-CNN 단독 아키텍처'로 메인 엔진을 경량화하여 임상 활용 최소 기준인 AUROC 0.84 돌파 및 위양성(FP) 제어에 성공함. 이후 하이브리드 모델의 장점을 완전히 버리지 않고 펩타이드 길이 확장 스펙에 유연하게 대응하기 위한 스마트 라우팅 체계 구축 단계로 이동함.

# [대화 06] 길이 적응형 스마트 라우터(Smart Router) 및 길이 페널티 도입

## 📌 작업 목표

최종 결과물의 도메인 타겟 스펙을 '하이드로젤 표면 수식용'에서 복잡한 3차원 독립 입체 구조를 형성하는 '독립적 폴리펩타이드(길이 30 이상)' 상태로 상향 조정함에 따라 발생하는 모델 성능의 한계를 해결한다. 서열 길이에 따라 최적의 신경망 엔진을 지능적으로 분기하고 무제한적인 합성 비용 상승을 막는 것을 목표로 한다.

## 🔧 주요 변경 사항 / 시도한 접근법

- **동적 라우팅 아키텍처(Smart Router) 구현:** 사용자가 입력하거나 유전 알고리즘이 생성하는 서열의 길이를 실시간 측정하여 최적의 부품으로 분배하는 하이브리드 동적 분기 모듈 도입.
    
    - $Length \le 15$: 오버피팅 위험이 적고 국소 패턴 스캔에 최적화된 **1D-CNN Only 엔진**으로 강제 라우팅.
        
    - $Length > 15$: 입체 접힘(Folding)에 따른 장거리 문맥 파악이 요구되는 **Hybrid (CNN+Bi-LSTM) 엔진**으로 정밀 라우팅.
        
- **비용 함수 내 길이 페널티(Length Penalty) 항 신설:** 유전 알고리즘의 세대 교체 중 서열이 무한정 길어져 실제 화학적 합성 단가가 천문학적으로 치솟는 리스크를 통제하기 위해, 타겟 최대 길이를 초과하는 개체에게 기하급수적인 벌점을 가산하는 제약 항 결합.
    

## 💻 핵심 코드 스니펫 (있는 경우)

Python

```
class LengthAdaptiveSmartRouter:
    """펩타이드 잔기 길이에 따라 최적의 예측 엔진을 동적으로 할당하는 라우터"""
    def __init__(self, cnn_engine, hybrid_engine):
        self.cnn_only = cnn_engine
        self.hybrid_core = hybrid_engine

    def route_sequence(self, sequence_string):
        seq_len = len(sequence_string)
        # 길이 특성에 따른 분기 라우팅 수행 (공학적 효율성 및 정확도 동시 확보)
        if seq_len <= 15:
            return self.cnn_only.predict_probability(sequence_string)
        else:
            return self.hybrid_core.predict_probability(sequence_string)

# 유전 알고리즘 비용 함수 내 길이 페널티 조건문 반영
def evaluate_length_constrained_cost(seq, base_cost, max_len=15, penalty_coef=0.5):
    current_len = len(seq)
    length_penalty = 0.0
    if current_len > max_len:
        # 길이를 초과한 만큼 비용 점수를 가산하여 GA 풀에서 자연 도태 유도
        length_penalty = (current_len - max_len) * penalty_coef
    return base_cost + length_penalty
```

## ❌ 실패하거나 버려진 시도 (있는 경우)

- **단일 고정형 신경망 모델 유지 기획 폐기:** 단일 CNN으로만 전체 시스템을 구성하려 했으나 독립형 폴리펩타이드의 입체적 맥락을 잡아내지 못하는 생물학적 한계에 부딪혔고, 그렇다고 전체 파이프라인을 하이브리드로 고정하자니 짧은 서열에서의 성능 저하와 연산 오버헤드가 발생하여 양자택일 방식의 고정형 아키텍처 기획을 전면 취소함.
